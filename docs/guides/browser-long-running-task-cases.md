# Machi 长程任务前端验收用例

给 QA 直接使用：每条用例包含可复制粘贴的指令、预期行为和判定标准。

## TC-01 长任务进度可见性

粘贴到 Machi：
```
帮我在当前agenticx仓库做一次全面排查：列出3个高风险点，每个点给复现步骤和修复建议，最后生成markdown报告保存到 /tmp/agx_audit_report.md
```

预期：
- 过程中持续出现工具调用块（file_read、bash_exec、grep_search 等）。
- 不会长时间只显示"正在输入"而无任何实质输出。
- 最终 `/tmp/agx_audit_report.md` 文件存在且有内容。

判定：能看到分阶段进展 = PASS；长时间空转或直接给结论无过程 = FAIL。

---

## TC-02 中断后恢复

粘贴到 Machi：
```
对agenticx仓库的runtime目录逐文件分析架构，给出依赖关系图和改进建议，保存到 /tmp/runtime_analysis.md
```

操作：等 Machi 开始执行工具调用后（至少看到 2~3 个工具结果），强制退出 App（Cmd+Q）。重启后回到该会话。

预期：
- 重启后消息列表保留中途已产出的工具调用和结果。
- 追问"继续上次的分析任务"后，Machi 基于已有进度继续，不从零重来。

判定：历史保留且可续跑 = PASS；消息丢失或完全重来 = FAIL。

---

## TC-03 失败提示可操作

粘贴到 Machi：
```
读取 /nonexistent/path/impossible_file.txt 的内容，然后基于内容生成总结报告
```

预期：
- 明确报错"文件不存在"或等价提示。
- 不会假装读到了内容并编造总结。
- 提供可操作建议（如"请确认路径"）。

判定：报错清晰且不编造 = PASS；假装成功或无提示卡死 = FAIL。

## TC-04 Token 预算收敛

粘贴到 Machi：
```
请逐文件详细分析agenticx/runtime/目录下所有Python文件，每个文件都要列出：函数签名、职责说明、潜在问题、改进建议。不要省略任何文件。
```

预期：
- 分析到一定量后，Machi 开始主动收敛（如"剩余文件概要列出"而非继续逐行展开）。
- 不会在没有任何提示的情况下突然中断。
- 若触发预算限制，提示文案包含"token"或"预算"相关说明。

判定：有收敛过渡而非硬停 = PASS；无征兆突然中断 = FAIL。

分析agenticx/runtime/agent_runtime.py的架构，生成一份包含类图和函数调用关系的技术文档，保存到 /tmp/agent_runtime_doc.md

预期：

- 若文件未生成，不应显示"已完成"，而是提示未产出并建议重试。

判定：文件存在且有内容 = PASS；显示完成但文件不存在 = FAIL。

## TC-06 重试一致性

操作：对 TC-03 或 TC-05 中失败的消息，连续点击"重试"按钮 3 次。

预期：
- 每次重试有独立的新一轮输出。
- 不出现旧结果与新结果重复拼接。
- 不出现状态混乱（如同时显示成功和失败）。

判定：每次重试独立干净 = PASS；出现拼接或串状态 = FAIL。

---

## TC-07 子智能体/分身执行透明

粘贴到 Machi（需有分身配置）：
```
@信长 对agenticx仓库进行安全审计，列出前3个高危问题，保存报告到 /tmp/security_audit.md
```

（如无"信长"分身，可改为任意已配置分身名）

预期：
- 可看到分身启动、执行中、完成的状态变化。
- 主对话收到分身执行结果汇总。
- 若分身失败，主对话有失败说明而非静默。

判定：分身过程可见且结果回传 = PASS；静默或丢失结果 = FAIL。

---

## TC-08 子智能体失败升级

粘贴到 Machi（需有分身配置）：
```
@信长 读取 /nonexistent/impossible_path/data.json 并基于内容生成分析报告保存到 /tmp/fail_test.md
```

预期：
- 分身首次失败后自动重试（前端可见 retry 过程）。
- 多次失败后熔断，主对话明确报告"执行失败"并附原因摘要。
- 不会无限重试也不会静默吞掉失败。

判定：重试可见 + 熔断有说明 = PASS；静默失败或无限循环 = FAIL。

## TC-09 多窗格隔离

操作：打开两个聊天窗格（Pane A / Pane B）。

Pane A 粘贴：
```
逐文件分析agenticx/runtime/目录，给出完整架构梳理
```

Pane B 粘贴：
```
你好，今天天气怎么样？
```

预期：
- Pane B 秒回，不被 Pane A 的长任务阻塞。
- Pane A 的工具调用结果不出现在 Pane B。
- 两边的模型标识、token 计数各自独立。

判定：互不干扰 = PASS；串消息或互相阻塞 = FAIL。

## TC-11 切会话后状态保持

操作：
1. 发起 TC-01 的长任务。
2. 执行中切换到另一个会话聊几句。
3. 切回原会话。

预期：
- 原会话内容完整保留。
- 不出现"正在初始化"卡死。
- 若任务已完成，结果正常展示。

判定：切回后状态正确 = PASS；内容丢失或卡死 = FAIL。

## TC-12 确认超时自动继续

粘贴到 Machi：
```
在 /tmp/ 目录下创建一个测试文件 agx_confirm_test.txt，内容写"hello world"
```

操作：当弹出确认请求时，不做任何操作，等待超时（默认 120 秒，可通过环境变量 AGX_CONFIRM_TIMEOUT_SEC 调小到 10 秒加速测试）。

预期：
- 超时后自动继续执行（默认 approve）。
- 不会永久卡在"等待确认"。

判定：超时后自动继续 = PASS；永久卡住 = FAIL。

## 验收门槛

- TC-01 ~ TC-05、TC-12：必须全部 PASS（核心稳定性）。
- TC-06 ~ TC-11：允许 1~2 项非阻断问题，但不得出现数据丢失或串会话。
- TC-13 ~ TC-24：长程复杂任务，每条至少产出可验证文件，允许内容质量波动但不允许假完成。
